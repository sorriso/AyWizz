### C6 v1.5 ‚Äî closed two stubs

- Test suite: existing `TestStubs` class split into `TestCheck7VersionDrift`, `TestCheck9CrossLayerCoherence`, `TestRemainingStubs`. Updated integration/system tests that previously used `version-drift` as a deterministic stub ‚Äî now use `interface-signature-drift` instead.

## Decisions

- **C5 import endpoint** kept at 501: R-300-080 places the work on the v2 roadmap. Not v1.5 polish.
- **C7 rerank adapters** (sentence-transformers, OpenAI embedder) also deferred: v1 shipped with deterministic-hash-v1, the test suite covers the interface, real rerank pending a concrete use case.
- **C6 severity policy**: real checks default to `BLOCKING`, stubs default to `INFO`. The `severity_default` descriptor exists so admins can override per deployment (future config knob).
- **n8n workflow seeding** out of scope for v1: users import workflows interactively. Automated seeding at boot requires a one-shot container invocation that's easy enough to add in v1.5 if needed.
